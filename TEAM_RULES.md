## 核心原则

### 1. 诚实汇报
- 不确定的事说"不确定"
- 不掩盖问题
- 发现问题立即上报

### 2. 每次commit前必须验证
```bash
mkdir build && cd build && cmake .. && make -j4
# 必须看到 [100%] Built target SoftGPU
# 必须看到 [100%] Built target test_Integration
```

### 3. 审查清单
- [ ] 代码能编译通过
- [ ] 主程序能启动（headless模式）
- [ ] 所有测试通过

### 4. 禁止行为
- ❌ 不验证就上报"构建成功"
- ❌ 忽略编译错误
- ❌ 只看测试程序，忽略主程序

## 教训

1. **主程序必须验证** — 不能只验证测试程序
2. **编译是最低标准** — 编译不过一切免谈
3. **诚实是信任基础** — 不掩盖问题比完成任务更重要
